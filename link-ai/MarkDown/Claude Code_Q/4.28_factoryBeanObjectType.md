# Spring Boot `factoryBeanObjectType` 错误解决方案

## 问题描述

在运行Spring Boot应用程序时遇到以下错误：
```
java.lang.IllegalArgumentException: Invalid value type for attribute 'factoryBeanObjectType': java.lang.String
```

错误发生在Spring上下文初始化阶段，具体在`PostProcessorRegistrationDelegate.invokeBeanFactoryPostProcessors`方法中。

## 错误分析

通过分析错误堆栈和项目配置，确定该问题是由于Spring Boot 3.2.0版本中的一个bug导致的。该bug在某些自动配置处理过程中，错误地将`factoryBeanObjectType`属性设置为String类型而不是期望的Class类型。

## 解决方案

### 1. 降级Spring Boot版本（主要解决方案）

修改`/e/private_project/LinkAI/hmall/pom.xml`文件：

**修改前：**
```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.0</version>
    <relativePath/>
</parent>
```

**修改后：**
```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.1.5</version>
    <relativePath/>
</parent>
```

### 2. 更新Spring Cloud版本以保持兼容性

**修改前：**
```xml
<spring-cloud.version>2023.0.0</spring-cloud.version>
```

**修改后：**
```xml
<spring-cloud.version>2022.0.4</spring-cloud.version>
```

### 3. 临时移除可能导致问题的依赖

修改`/e/private_project/LinkAI/hmall/hm-service/pom.xml`文件：

- 注释掉Spring AI相关依赖
- 注释掉Redis依赖

### 4. 修复编译错误

修改`/e/private_project/LinkAI/hmall/hm-service/src/main/java/com/hmall/config/AIServiceConfig.java`：

简化AI服务配置类，移除对Spring AI的依赖引用，避免编译错误。

## 技术总结

1. **根本原因**：Spring Boot 3.2.0中的自动配置处理bug
2. **解决方案**：降级到稳定的Spring Boot 3.1.5 LTS版本
3. **替代方案**：可以尝试排除特定的自动配置类，但降级是最简单可靠的解决方案
4. **后续建议**：可以逐步重新启用被注释的依赖，测试兼容性，或等待Spring Boot后续版本修复此问题
