# 问题总结：chat_session 表中 user_id 始终为 0 的问题

## 问题现象

登录用户为 Jack（user_id = 1），但 `chat_session` 表中记录的 `user_id` 始终为 0，且 `chat_history` 表中也查不到对应的会话历史记录。

## 问题根因分析

### 1. 后端：`ChatSessionServiceImpl.createSession` 中 userId 为 null 时默认设为 0

```java
// ChatSessionServiceImpl.java 第 57-59 行
if (userId == null) {
    userId = 0L; // 未登录用户使用默认值
}
```

当 `userId` 为 `null` 时，代码直接将其设为 `0L`，而不是抛出异常或要求前端提供正确的 userId。这导致即使前端没有正确传递 userId，会话也能被创建，但 user_id 为 0。

### 2. 前端：`agent-chat.js` 中 `loadExamples()` 错误调用 `POST /session`

```javascript
// 修改前：agent-chat.js loadExamples()
async function loadExamples() {
    try {
        const response = await fetch(CONFIG.API_BASE + CONFIG.SESSION_ENDPOINT, {
            method: 'POST',  // ❌ 错误！应该用 GET
            headers: { 'Content-Type': 'application/json' },
        });
        // ...
    }
}
```

`loadExamples()` 的本意是**加载模版问题（示例问题）**，但却调用了 `POST /session`（创建会话接口），且**没有传递 `authorization` 请求头**。这导致：

- 后端 `SessionController.createSession` 中 `token` 为 null
- `userId` 解析失败，最终为 null
- `createSession` 将 userId 设为 0，创建了一条 user_id = 0 的垃圾会话记录

### 3. 前端：`agent-chat.js` 中 `sendMessage()` 前端自己生成 sessionId

```javascript
// 修改前：agent-chat.js sendMessage()
if (!state.currentSessionId) {
    const sessionId = generateSessionId();  // ❌ 前端自己生成，不是从后端获取
    // ...
    state.currentSessionId = sessionId;
}
```

`sendMessage()` 在没有 `currentSessionId` 时，直接在前端用 `generateSessionId()` 生成一个随机 sessionId，而不是调用后端 `POST /session` 接口获取后端生成的 sessionId。这导致：

- 后续 `update` 时用这个前端生成的 sessionId 去数据库查询，当然查不到
- 会话记录和消息记录无法关联到正确的后端数据

## 解决措施

### 措施 1：后端 `ChatSessionServiceImpl.createSession` — userId 为 null 时抛出异常

```java
// ChatSessionServiceImpl.java
if (userId == null) {
    throw new IllegalArgumentException("请先登录后再操作");
}
```

不再默认设为 0，而是抛出异常，强制要求前端必须传递正确的 userId。

### 措施 2：后端 `SessionController.createSession` — 支持从请求参数获取 userId

```java
// SessionController.java
@PostMapping
public SessionVO createSession(@RequestParam(value = "n", defaultValue = "3") Integer n,
                               @RequestParam(value = "userId", required = false) Long paramUserId,
                               HttpServletRequest request) {
    // 1. 尝试从 token 解析 userId
    Long userId = null;
    String token = request.getHeader("authorization");
    if (token != null && !token.isEmpty()) {
        try {
            userId = jwtTool.parseToken(token);
        } catch (Exception e) {
            log.warn("token 解析失败: {}", e.getMessage());
        }
    }

    // 2. 如果 token 解析失败，尝试从请求参数获取 userId（前端降级方案）
    if (userId == null && paramUserId != null) {
        userId = paramUserId;
    }

    // 3. 如果还是没有 userId，尝试从 UserContext 获取
    if (userId == null) {
        userId = UserContext.getUser();
    }

    return this.chatSessionService.createSession(n, userId);
}
```

新增 `@RequestParam("userId")` 参数，当前端没有 token 时，可以通过 `?userId=1` 直接传递 userId。

### 措施 3：前端 `agent-chat.js` `loadExamples()` — 改为调用 `GET /session/hot`

```javascript
// 修改后：agent-chat.js loadExamples()
async function loadExamples() {
    try {
        const response = await fetch(CONFIG.API_BASE + CONFIG.SESSION_ENDPOINT + '/hot?n=3', {
            method: 'GET',  // ✅ 改为 GET 请求
        });
        if (!response.ok) throw new Error(`HTTP ${response.status}`);
        const data = await response.json();
        state.examples = data || [];
    } catch (err) {
        console.error('[LinkAI] 加载示例问题失败:', err);
    }
}
```

改为调用 `GET /session/hot?n=3`，只获取模版问题，不再意外创建会话记录。

### 措施 4：前端 `agent-chat.js` `sendMessage()` — 改为调用后端 API 获取 sessionId

```javascript
// 修改后：agent-chat.js sendMessage()
if (!state.currentSessionId) {
    // 调用后端 API 创建会话，获取后端生成的 sessionId
    try {
        const token = sessionStorage.getItem('token') || '';
        const userInfoStr = sessionStorage.getItem('user-info');
        let userId = null;
        if (userInfoStr) {
            try {
                const userInfo = JSON.parse(userInfoStr);
                userId = userInfo.userId || null;
            } catch (e) {}
        }

        const headers = { 'Content-Type': 'application/json' };
        if (token) {
            headers['authorization'] = token;
        }
        let url = CONFIG.API_BASE + CONFIG.SESSION_ENDPOINT + '?n=3';
        if (userId) {
            url += '&userId=' + userId;  // ✅ 传递 userId 参数
        }
        const sessionResp = await fetch(url, {
            method: 'POST',
            headers: headers,
        });
        if (sessionResp.ok) {
            const sessionData = await sessionResp.json();
            const sessionId = sessionData.sessionId;  // ✅ 使用后端生成的 sessionId
            // ...
            state.currentSessionId = sessionId;
        } else {
            throw new Error(`HTTP ${sessionResp.status}`);
        }
    } catch (err) {
        console.error('[LinkAI] 创建会话失败:', err);
        // 降级方案：前端自己生成 sessionId
        const sessionId = generateSessionId();
        // ...
        state.currentSessionId = sessionId;
    }
}
```

- 先尝试调用后端 `POST /session` 创建会话，获取后端生成的 `sessionId`
- 从 `sessionStorage` 中获取 `token` 和 `userId`，传递给后端
- 如果后端创建失败，降级为前端自己生成 sessionId
