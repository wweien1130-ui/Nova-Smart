# MyBatis-Plus 版本分水岭：PaginationInnerInterceptor 依赖变迁

## 一、问题现象

在 `hmall/hm-common/src/main/java/com/hmall/common/config/MyBatisConfig.java` 中：

```java
import com.baomidou.mybatisplus.extension.plugins.inner.PaginationInnerInterceptor;
```

无论切换哪个版本的 MyBatis-Plus，IDE 始终报错：**无法解析符号 'PaginationInnerInterceptor'**。

运行时也抛出：

```
java.lang.ClassNotFoundException: com.baomidou.mybatisplus.extension.plugins.inner.PaginationInnerInterceptor
```

---

## 二、版本分水岭：3.5.9

MyBatis-Plus 在 **3.5.9** 版本做了一次重要的模块重构，这是关键的分水岭：

| 版本范围 | PaginationInnerInterceptor 所在模块 | 是否需要额外依赖 |
|---------|-----------------------------------|----------------|
| **≤ 3.5.8** | `mybatis-plus-extension` | 不需要，extension 已包含 |
| **≥ 3.5.9** | `mybatis-plus-jsqlparser` | **需要**，被独立拆分出来 |

### 为什么要在 3.5.9 拆分？

1. **解耦 JSqlParser 依赖**：`PaginationInnerInterceptor` 内部依赖 `JSqlParser`（SQL 解析库），将分页拦截器拆分到独立模块后，不使用分页功能的项目无需引入 JSqlParser，减少依赖体积。

2. **模块职责清晰化**：
   - `mybatis-plus-extension` → 核心扩展功能
   - `mybatis-plus-jsqlparser` → 需要 SQL 解析能力的组件（分页、多租户等）

3. **避免版本冲突**：JSqlParser 自身有多个大版本，独立模块便于管理和升级。

---

## 三、项目依赖分析

### 当前项目配置（hmall/pom.xml）

```xml
<mybatis-plus.version>3.5.12</mybatis-plus.version>
```

### hm-common/pom.xml（修复前）

```xml
<!-- 只有 extension，缺少 jsqlparser -->
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-extension</artifactId>
    <version>${mybatis-plus.version}</version>
    <scope>provided</scope>
</dependency>
```

### hm-service/pom.xml

```xml
<!-- 引入了 mybatis-plus-spring-boot3-starter -->
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-spring-boot3-starter</artifactId>
</dependency>
```

### 依赖传递关系

```
mybatis-plus-spring-boot3-starter (3.5.12)
  └── mybatis-plus (3.5.12)
        ├── mybatis-plus-extension (3.5.12)     ← 不含 PaginationInnerInterceptor
        └── mybatis-plus-jsqlparser (3.5.12)    ← 含 PaginationInnerInterceptor（可选依赖？）
```

> **关键点**：`mybatis-plus-spring-boot3-starter` 虽然传递依赖了 `mybatis-plus-jsqlparser`，但 `hm-common` 是独立编译的模块，它直接依赖的是 `mybatis-plus-extension`（scope=provided），并没有直接或间接依赖 `mybatis-plus-jsqlparser`，所以编译时找不到 `PaginationInnerInterceptor`。

---

## 四、解决方案

### 修复：在 hm-common/pom.xml 中添加 mybatis-plus-jsqlparser

```xml
<!-- MyBatis-Plus JSqlParser (3.5.9+ 必需，PaginationInnerInterceptor 移至此模块) -->
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-jsqlparser</artifactId>
    <version>${mybatis-plus.version}</version>
</dependency>
```

> **注意 scope**：不能设为 `provided`，因为 `hm-service` 依赖 `hm-common` 时，`provided` 范围的依赖不会被传递打包进可执行 JAR，导致运行时 `ClassNotFoundException`。

### 修复前后对比

| 对比项 | 修复前 | 修复后 |
|-------|-------|-------|
| IDE 报错 | ❌ 无法解析符号 PaginationInnerInterceptor | ✅ 正常识别 |
| 编译 | ✅ 通过（IDE 缓存） | ✅ 通过 |
| 运行时 | ❌ ClassNotFoundException | ✅ 正常加载 |
| JAR 包内容 | 不含 mybatis-plus-jsqlparser | ✅ 包含 mybatis-plus-jsqlparser-3.5.12.jar |

---

## 五、MyBatis-Plus 各模块功能一览

| 模块 | 功能 | 3.5.9+ 是否必需 |
|------|-----|----------------|
| `mybatis-plus` | 核心模块，提供基础 CRUD、条件构造器等 | ✅ 必需 |
| `mybatis-plus-extension` | 扩展模块，提供分页、多租户、乐观锁等拦截器接口 | ✅ 必需 |
| `mybatis-plus-jsqlparser` | SQL 解析模块，包含 PaginationInnerInterceptor 等需要 JSqlParser 的实现类 | ⚠️ 使用分页/多租户时需要 |
| `mybatis-plus-spring-boot3-starter` | Spring Boot 3 自动配置 starter | ✅ Spring Boot 3 项目必需 |
| `mybatis-plus-generator` | 代码生成器 | 可选 |
| `mybatis-plus-annotation` | 注解模块 | 通常被核心模块传递依赖 |

---

## 六、总结

1. **MyBatis-Plus 3.5.9 是一个分水岭**，`PaginationInnerInterceptor` 从 `mybatis-plus-extension` 移到了 `mybatis-plus-jsqlparser`
2. 使用 **3.5.9+** 版本时，如果代码中直接引用了 `PaginationInnerInterceptor`，必须显式添加 `mybatis-plus-jsqlparser` 依赖
3. 依赖的 **scope 必须为 compile**（默认），不能是 provided，否则运行时找不到类
4. 如果降级到 **3.5.8 或更低**，则不需要 `mybatis-plus-jsqlparser`，但会失去后续版本的 bug 修复和新特性
