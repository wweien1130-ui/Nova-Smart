## 二、Maven 资源复制失败

### 问题描述

`static` 目录不存在，Maven 无法复制文件。

### 解决方案
手动创建 `static` 目录：
```bash
mkdir "E:\private_project\LinkAI\hmall\agent\src\main\resources\static"
```
然后将 `chat.html` 复制进去。

## 三、前端请求 404（接口路径问题）

### 问题描述
前端发送消息时报错：
```
Failed to load resource: the server responded with a status of 404 (Not Found)
```

但通过 HTTP 客户端直接请求 `POST http://localhost:8081/agent/chat` 却能正常返回。

### 原因
前端通过 `file://` 协议打开 `chat.html`，请求使用的是相对路径 `/agent/chat`，实际变成了 `file:///agent/chat`，所以 404。

### 解决方案
将 fetch 的 URL 从相对路径改为完整路径：
```javascript
// 修改前
const response = await fetch('/agent/chat', { ... });

// 修改后
const response = await fetch('http://localhost:8081/agent/chat', { ... });
```

## 四、跨域问题（CORS）

### 问题描述
前端通过 `file://` 协议访问，请求 `http://localhost:8081` 时跨域。

### 解决方案
在 `AgentController` 上添加 `@CrossOrigin` 注解：
```java
@RestController
@RequestMapping("/agent")
@RequiredArgsConstructor
@CrossOrigin(origins = "*")  // 允许所有来源跨域
public class AgentController {
    // ...
}
```

## 五、AI Agent 工具（Tool）未生效

### 问题描述
`ItemSearchTool` 中定义了 `@Tool` 注解的 `searchItems` 和 `getItemDetail` 方法，但 AI 模型始终不调用这些工具，控制台没有任何打印输出。即使传入数据库中真实存在的商品 ID 查询详情，也搜不到任何商品信息。

### 原因
`ChatClient` 在构建时没有通过 `.defaultTools()` 注册工具，AI 模型根本不知道有这些工具可用。

### 解决方案
在 `AIServiceConfig.java` 的 `chatClient` Bean 方法中：
1. 注入 `ItemSearchTool` 参数
2. 添加 `.defaultTools(itemSearchTool)` 注册工具

```java
@Bean
public ChatClient chatClient(ChatModel dashScopeChatModel, ChatMemory chatMemory,
                             ItemSearchTool itemSearchTool) {
    return ChatClient.builder(dashScopeChatModel)
            .defaultSystem("你是一个智能购物助手，名字叫\"小惠\"")
            .defaultTools(itemSearchTool)  // ← 关键：注册工具
            .defaultAdvisors(
                    MessageChatMemoryAdvisor.builder(chatMemory).build()
            )
            .defaultOptions(
                    DashScopeChatOptions.builder().withModel("deepseek-v4-pro").build()
            )
            .build();
}
```

### 验证方式
重启后发送"帮我搜索衣服"，控制台应打印：
```
========== 调用了 searchItems，关键词: 衣服 ==========
```

## 六、MP 模糊查询性能问题（讨论中）

### 问题描述
当前 `ItemSearchTool` 使用 MyBatis-Plus 的 `LIKE %keyword%` 做模糊查询：
```java
Page<Item> result = itemService.lambdaQuery()
        .like(Item::getName, keyword)
        .eq(Item::getStatus, 1)
        .page(new Page<>(1, 10));
```

存在以下问题：
- `LIKE %keyword%` 无法走索引，全表扫描
- 不支持分词（搜"笔记本电脑"搜不到"笔记本"）
- 不支持相关性排序
- 不支持关键词高亮
- 商品表没有分类字段

### 讨论方案
引入 Elasticsearch 替代 MP 的模糊查询：
1. Docker 搭建 ES + Kibana，配置 IK 分词器
2. 数据同步方案：
   - **方案 A**：Logstash 同步（零代码，配置即用）
   - **方案 B**：代码同步（更灵活，可控性高）
3. 改造 `ItemSearchTool`，使用 ES 的 `multiMatchQuery` 进行分词搜索
