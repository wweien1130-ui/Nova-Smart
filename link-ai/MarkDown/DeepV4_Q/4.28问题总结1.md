## 1. 版本演进路线

```
Spring Boot 3.1.5 (原始稳定版)
    │
    ├── 尝试升级 → 3.2.0  ❌ (factoryBeanObjectType Bug)
    │
    ├── 尝试升级 → 3.2.12 ✅ (但 MyBatis-Plus 兼容性问题)
    │
    └── 最终方案 → 3.1.5 (回退稳定) 或 3.2.12 + mybatis-plus-jsqlparser
```

| 版本 | 状态 | 说明 |
|------|------|------|
| 3.1.5 | ✅ 稳定 | 原始版本，所有功能正常 |
| 3.2.0 | ❌ 启动失败 | `factoryBeanObjectType` 类型错误 Bug |
| 3.2.12 | ⚠️ 需额外配置 | 需配合 MyBatis-Plus 3.5.9+ 及 jsqlparser 模块 |

## 2. 核心变更一：Jakarta EE 迁移（改动最大）

### 2.1 变更概述

**这是从 Spring Boot 2.x → 3.x 最大的架构变更**，影响面最广，涉及项目中几乎所有 Java 文件。

### 2.2 变更内容

| 旧包 (javax.*) | 新包 (jakarta.*) | 影响范围 |
|----------------|------------------|---------|
| `javax.servlet.*` | `jakarta.servlet.*` | Web 层所有类 |
| `javax.validation.*` | `jakarta.validation.*` | 参数校验注解 |
| `javax.persistence.*` | `jakarta.persistence.*` | JPA 相关 |

### 2.3 受影响文件清单

| 文件 | 变更内容 |
|------|---------|
| `WebUtils.java` | `javax.servlet.http.HttpServletRequest/Response` → `jakarta.servlet.*` |
| `CookieBuilder.java` | `javax.servlet.http.Cookie` → `jakarta.servlet.http.Cookie` |
| `LoginInterceptor.java` | `javax.servlet` → `jakarta.servlet` |
| `CartController.java` | `javax.validation.Valid` → `jakarta.validation.Valid` |
| `HelloController.java` | `javax.servlet.HttpServletRequest` → `jakarta.servlet.*` |
| `LoginFormDTO.java` | `javax.validation.constraints.NotNull` → `jakarta.validation.*` |
| `PayApplyDTO.java` | `javax.validation.constraints` → `jakarta.validation.*` |
| `PayOrderFormDTO.java` | `javax.validation.constraints.NotNull` → `jakarta.validation.*` |
| `PageQuery.java` | `javax.validation.constraints.Min` → `jakarta.validation.*` |

### 2.4 依赖变更

```xml
<!-- 旧：Java EE -->
<dependency>
    <groupId>javax.servlet</groupId>
    <artifactId>javax.servlet-api</artifactId>
</dependency>

<!-- 新：Jakarta EE -->
<dependency>
    <groupId>jakarta.servlet</groupId>
    <artifactId>jakarta.servlet-api</artifactId>
    <version>6.0.0</version>
    <scope>provided</scope>
</dependency>
```

### 2.5 影响评估

- **影响范围**：⭐⭐⭐⭐⭐（全项目）
- **改动量**：⭐⭐⭐⭐⭐（10+ 个 Java 文件）
- **难度**：⭐⭐（机械替换，但容易遗漏）
- **风险**：⭐⭐⭐（遗漏会导致编译失败）

## 3. 核心变更二：Spring Boot 3.2.x 的 factoryBeanObjectType Bug

### 3.1 问题现象

```
java.lang.IllegalArgumentException: Invalid value type for attribute 'factoryBeanObjectType': java.lang.String
```

### 3.2 根本原因

Spring Boot **3.2.0** 版本在自动配置处理过程中存在 Bug，错误地将 `factoryBeanObjectType` 属性设置为 `String` 类型而不是期望的 `Class` 类型。

### 3.3 触发条件

- Spring Boot 3.2.0
- 项目中使用了 MyBatis-Plus（或其他通过 FactoryBean 创建 Bean 的框架）
- 自动配置类处理时触发

### 3.4 解决方案

| 方案 | 操作 | 评价 |
|------|------|------|
| **方案1：降级** | 回退到 Spring Boot 3.1.5 | ✅ 最简单可靠 |
| **方案2：升级到 3.2.12+** | 使用修复了该 Bug 的版本 | ✅ 保留高版本特性 |
| **方案3：排除自动配置** | 排除特定自动配置类 | ❌ 复杂且不可靠 |

### 3.5 影响评估

- **影响范围**：⭐⭐（仅启动阶段）
- **改动量**：⭐（仅改版本号）
- **难度**：⭐⭐⭐（需要定位根因）
- **风险**：⭐⭐⭐⭐（降级可能丢失新特性）

## 4. 核心变更三：MyBatis-Plus 3.5.9 模块重构

### 4.1 变更概述

MyBatis-Plus 在 **3.5.9** 版本做了模块重构，这是配合 Spring Boot 3.2+ 的关键变更。

### 4.2 版本分水岭

| 版本范围 | PaginationInnerInterceptor 所在模块 | 是否需要额外依赖 |
|---------|-----------------------------------|----------------|
| **≤ 3.5.8** | `mybatis-plus-extension` | ❌ 不需要 |
| **≥ 3.5.9** | `mybatis-plus-jsqlparser` | ✅ **必须添加** |

### 4.3 重构原因

1. **解耦 JSqlParser 依赖**：分页拦截器内部依赖 JSqlParser（SQL 解析库），拆分后不使用分页功能的项目无需引入
2. **模块职责清晰化**：
   - `mybatis-plus-extension` → 核心扩展功能
   - `mybatis-plus-jsqlparser` → 需要 SQL 解析能力的组件
3. **避免版本冲突**：JSqlParser 独立模块便于管理和升级

### 4.4 解决方案

在 `hm-common/pom.xml` 中添加：

```xml
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-jsqlparser</artifactId>
    <version>${mybatis-plus.version}</version>
    <!-- 注意：scope 不能设为 provided，否则运行时 ClassNotFoundException -->
</dependency>
```

### 4.5 各模块功能一览

| 模块 | 功能 | 3.5.9+ 是否必需 |
|------|-----|----------------|
| `mybatis-plus` | 核心模块，基础 CRUD、条件构造器 | ✅ 必需 |
| `mybatis-plus-extension` | 扩展模块，分页、多租户、乐观锁接口 | ✅ 必需 |
| `mybatis-plus-jsqlparser` | SQL 解析模块，含 PaginationInnerInterceptor 实现 | ⚠️ 使用分页时需要 |
| `mybatis-plus-spring-boot3-starter` | Spring Boot 3 自动配置 starter | ✅ Spring Boot 3 必需 |
| `mybatis-plus-generator` | 代码生成器 | 可选 |
| `mybatis-plus-annotation` | 注解模块 | 通常被传递依赖 |

### 4.6 影响评估

- **影响范围**：⭐⭐⭐（MyBatis-Plus 用户）
- **改动量**：⭐（仅加一个依赖）
- **难度**：⭐⭐⭐（需要理解模块拆分逻辑）
- **风险**：⭐⭐（依赖 scope 设置错误会导致运行时异常）

## 5. 核心变更四：API 文档工具链迁移

### 5.1 变更概述

Spring Boot 3.x 不再支持旧的 SpringFox (Swagger 2)，需要迁移到 springdoc-openapi (OpenAPI 3)。

### 5.2 依赖变更

```xml
<!-- 旧：SpringFox / knife4j-openapi2（不兼容 Spring Boot 3.x） -->
<dependency>
    <groupId>com.github.xiaoymin</groupId>
    <artifactId>knife4j-openapi2-spring-boot-starter</artifactId>
</dependency>

<!-- 新：springdoc-openapi（兼容 Spring Boot 3.x） -->
<dependency>
    <groupId>org.springdoc</groupId>
    <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
    <version>2.1.0</version>
</dependency>
```

### 5.3 配置变更

```yaml
# application.yaml 中需要放行的新路径
hm:
  auth:
    excludePaths:
      - /v3/api-docs/**     # OpenAPI 3 JSON 文档
      - /doc.html           # Knife4j UI
      - /swagger-ui/**      # Swagger UI
      - /swagger-resources/**
      - /webjars/**         # WebJars 资源
```

### 5.4 拦截器配置

```java
// MvcConfig.java 中确保放行规则优先级
@Override
public void addInterceptors(InterceptorRegistry registry) {
    // 优先使用配置文件中的排除路径
    List<String> excludePaths = authProperties.getExcludePaths();
    if (CollUtil.isNotEmpty(excludePaths)) {
        registration.excludePathPatterns(excludePaths);
    }
}
```

### 5.5 影响评估

- **影响范围**：⭐⭐（仅 API 文档相关）
- **改动量**：⭐⭐（依赖 + 配置 + 拦截器）
- **难度**：⭐⭐（标准迁移流程）
- **风险**：⭐⭐（路径放行遗漏会导致 404）

## 6. 核心变更五：Spring AI Alibaba 集成

### 6.1 版本兼容性

| Spring Boot 版本 | Spring AI Alibaba 版本 | 状态 |

| 3.1.5 | 不支持 | ❌ 需要 Spring Boot 3.2+ |
| 3.2.0 | 1.0.0.2 | ❌ factoryBeanObjectType Bug |

## 7. 版本兼容性矩阵

### 7.1 推荐版本组合

| 组件 | 低版本组合（稳定） | 高版本组合（最新） |
|------|------------------|------------------|
| **Spring Boot** | 3.1.5 | 3.2.12 |
| **Spring Cloud** | 2022.0.4 | 2023.0.x |
| **Spring Cloud Alibaba** | 2022.0.0.0 | 2022.0.0.0 |
| **MyBatis-Plus** | 3.5.5 | 3.5.12 |
| **MyBatis-Plus jsqlparser** | ❌ 不需要 | ✅ 必须添加 |
| **Spring AI Alibaba** | ❌ 不支持 | 1.0.0.2 |
| **springdoc-openapi** | 2.1.0 | 2.1.0 |
| **Jakarta Servlet API** | ❌ 不需要 | 6.0.0 |

### 7.2 版本对应关系速查

```
Spring Boot 3.1.x
  ├── javax.* → 仍可使用（但建议迁移）
  ├── MyBatis-Plus ≤ 3.5.8（无需 jsqlparser）
  └── Spring AI Alibaba → ❌ 不支持

Spring Boot 3.2.x
  ├── jakarta.* → 必须迁移
  ├── MyBatis-Plus ≥ 3.5.9（需要 jsqlparser）
  ├── Spring AI Alibaba → ✅ 支持
  └── 注意：3.2.0 有 factoryBeanObjectType Bug，建议 ≥ 3.2.12
```

## 8. 总结：改动最大排名

| 排名 | 变更项 | 影响文件数 | 改动量 | 难度 | 综合评分 |
|------|--------|-----------|-------|------|---------|
| 🥇 | **Jakarta EE 迁移** (javax → jakarta) | 10+ | ⭐⭐⭐⭐⭐ | ⭐⭐ | **最大** |
| 🥈 | **MyBatis-Plus 3.5.9 模块重构** | 1 (依赖配置) | ⭐ | ⭐⭐⭐ | **第二大** |
| 🥉 | **Spring Boot 3.2.0 Bug 处理** | 1 (版本号) | ⭐ | ⭐⭐⭐ | **第三大** |
| 4 | **API 文档工具链迁移** | 3+ | ⭐⭐ | ⭐⭐ | - |
| 5 | **Spring AI Alibaba 集成** | 2+ | ⭐ | ⭐⭐ | - |

### 关键结论

1. **改动最大的单项**：**Jakarta EE 迁移**（javax → jakarta），涉及 10+ 个文件，影响整个 Web 层和校验层
2. **最隐蔽的坑**：**MyBatis-Plus 3.5.9 模块拆分**，只改一个依赖配置，但排查难度大
3. **最致命的 Bug**：**Spring Boot 3.2.0 的 factoryBeanObjectType**，直接导致应用无法启动
4. **最佳实践**：升级 Spring Boot 时，建议**跳过 3.2.0**，直接使用 **3.2.12+**，并同步升级 MyBatis-Plus 到 **3.5.9+** 且添加 `mybatis-plus-jsqlparser` 依赖
