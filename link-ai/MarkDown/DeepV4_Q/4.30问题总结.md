## 二、问题汇总

### 问题 1：ChatSessionServiceImpl.createSession() 中 builder 结果未赋值

**现象**：调用 `POST /session` 返回 500，数据库插入失败。

**原因**：`ChatSessionServiceImpl.createSession()` 中，builder 构建的 `ChatSession` 对象没有赋值给变量，直接调用了 `super.save(chatSession)`，但 `chatSession` 是之前未赋值的变量。

**修复前代码**：
```java
ChatSession.builder()
        .sessionId(sessionVo.getSessionId())
        .userId(UserContext.getUser())
        .build();
super.save(chatSession);  // chatSession 是 null！
```

**修复后代码**：
```java
ChatSession chatSession = ChatSession.builder()
        .sessionId(sessionVo.getSessionId())
        .userId(userId)
        .creater(userId)
        .updater(userId)
        .build();
super.save(chatSession);
```

### 问题 2：UserContext.getUser() 返回 null 导致数据库插入失败

**现象**：`POST /session` 返回 500，数据库 `creater`/`updater` 字段插入 null 失败。

**原因**：请求没有带 JWT Token，`UserContext`（ThreadLocal）中没有用户信息，`getUser()` 返回 null。而 `creater` 和 `updater` 字段在数据库中非空且无默认值。

**修复**：添加 null 判断，未登录用户使用默认值 0L。

```java
Long userId = UserContext.getUser();
if (userId == null) {
    userId = 0L; // 未登录用户使用默认值
}
```

### 问题 4：ChatServiceImpl 中 systemPrompt 为 null 导致 "text cannot be null or empty"

**现象**：调用 `POST /chat` 返回 500，报错 `java.lang.IllegalArgumentException: text cannot be null or empty`。

**原因**：`ChatServiceImpl` 中 `systemPromptConfig.getChatSystemMessage().get()` 返回了 null。`SystemPromptConfig.init()` 中从 Nacos 读取 `system-chat-message.yaml` 配置，如果 Nacos 读取失败或配置不存在，`chatSystemMessage` 保持为 null。Spring AI 的 `.text(null)` 会抛出异常。

**修复**：添加 null 安全处理，使用默认提示词作为 fallback。

```java
String systemText = systemPromptConfig.getChatSystemMessage().get();
if (systemText == null || systemText.isBlank()) {
    systemText = "你是一个智能购物助手，名字叫\"小惠\"，运行在\"LinkAI 电商平台\"上。";
}
```

### 问题 5：CORS 跨域问题

**现象**：前端在 `localhost:63342`（IntelliJ 内置服务器）运行，后端在 `localhost:8080`，请求被浏览器拦截。

```
Access to fetch at 'http://localhost:8080/session' from origin 'http://localhost:63342' 
has been blocked by CORS policy
```

**原因**：`AgentController` 有 `@CrossOrigin(origins = "*")`，但 `ChatController` 和 `SessionController` 没有。

**修复**：给 `ChatController` 和 `SessionController` 添加 `@CrossOrigin(origins = "*")` 注解。

```java
@RestController
@RequestMapping("/chat")
@RequiredArgsConstructor
@CrossOrigin(origins = "*")  // ← 添加
public class ChatController { ... }

@RestController
@RequestMapping("/session")
@RequiredArgsConstructor
@CrossOrigin(origins = "*")  // ← 添加
public class SessionController { ... }
```

### 问题 6：前端请求路径与后端 Controller 不匹配

**现象**：前端 `chat.html` 请求的是 `POST /agent/chat`（`AgentController`），但需要改为请求 `POST /chat`（`ChatController`）。

**差异对比**：

| 对比项 | AgentController | ChatController |
|--------|----------------|----------------|
| 路径 | `/agent/chat` | `/chat` |
| 请求体 | `{ message, sessionId }` | `{ question, sessionId }` |
| 响应格式 | `Flux<String>`（纯文本流） | `Flux<ChatEventVo>`（SSE 格式） |
| 响应示例 | `"你好"` | `data: {"eventData":"你好","eventType":1001}\n\n` |

**修复**：
1. 请求地址改为 `http://localhost:8080/chat`
2. 请求参数改为 `{ question, sessionId }`
3. 新增 SSE 格式解析逻辑，逐行解析 `data: {...}\n\n`，只提取 `eventType === 1001` 的数据

```javascript
// SSE 解析逻辑
const lines = buffer.split('\n');
buffer = lines.pop() || '';
for (const line of lines) {
    if (line.startsWith('data:')) {
        const event = JSON.parse(line.substring(5).trim());
        if (event.eventType === 1001) {
            fullContent += event.eventData || '';
        }
    }
}
```

### 问题 7：模版问题一闪而过

**现象**：刷新页面时，模版问题在欢迎页上闪现一下立即消失。

**原因**：初始化时先执行 `loadExamples()` 加载模版问题（显示在欢迎页），然后立即执行 `createNewChat()` 创建对话，`switchToChat()` 把欢迎页隐藏了。

**修复**：去掉自动创建对话的逻辑，让欢迎页作为默认首页展示。

```javascript
// 修复前
loadExamples().then(() => {
    createNewChat();  // 立即创建对话，隐藏欢迎页
});

// 修复后
loadExamples();  // 只加载模版问题，不自动创建对话
```

### 问题 8：新建对话后模版问题不显示

**现象**：点击「新建」按钮后，消息区域为空，模版问题消失。

**原因**：`renderMessages()` 中当消息为空时，只显示了一个简单的空状态提示，没有渲染模版问题。

**修复**：
1. 在 `state` 中添加 `examples: []` 字段存储模版问题数据
2. `loadExamples()` 中将数据保存到 `state.examples`
3. 新增 `renderExamplesHtml()` 函数生成模版问题 HTML
4. `renderMessages()` 中消息为空时调用 `renderExamplesHtml()` 显示模版问题

```javascript
// state 中添加 examples
const state = {
    chats: new Map(),
    currentSessionId: null,
    isStreaming: false,
    abortController: null,
    examples: [],  // ← 新增
};

// loadExamples 中保存数据
state.examples = data.examples || [];

// renderMessages 中空消息时显示模版问题
if (chat.messages.length === 0) {
    const examplesHtml = renderExamplesHtml();
    messagesContainer.innerHTML = `
        <div style="...">
            <div>🤖</div>
            <div>LinkAI 智能购物助手</div>
            <div>试试以下问题 👇</div>
            ${examplesHtml}
        </div>
    `;
}
```

## 四、经验总结

### 4.1 Nacos 配置注意事项
1. **配置更新后需要重启应用**（除非使用 `@RefreshScope`）
2. **Nacos 动态配置**（如 `system-chat-message.yaml`）读取失败时要有 fallback 默认值
3. **YAML 缩进必须正确**，否则配置项可能解析失败

### 4.2 数据库操作注意事项
1. **MyBatis-Plus builder 模式**：builder 结果必须赋值给变量
2. **非空字段**：插入数据前确保所有非空字段都有值
3. **UserContext 依赖**：未登录状态下 `getUser()` 返回 null，需要做 null 处理

### 4.3 前后端联调注意事项
1. **CORS 跨域**：所有被前端跨域访问的 Controller 都需要加 `@CrossOrigin`
2. **SSE 格式解析**：`text/event-stream` 格式为 `data: {...}\n\n`，前端需要逐行解析
3. **请求参数匹配**：前端请求的字段名必须与后端 DTO 的字段名一致

### 4.4 前端状态管理注意事项
1. **初始化顺序**：异步加载数据（如模版问题）要在 DOM 渲染完成后执行
2. **状态共享**：全局数据（如模版问题）要存储在 state 中，供多个函数共用
3. **欢迎页 vs 聊天页**：切换时注意隐藏/显示对应的 DOM 元素
