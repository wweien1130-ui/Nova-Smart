# 5.3 问题总结

## 一、背景

在 LinkAI 项目中，Agent（智能购物助手）通过 Spring AI 的 `@Tool` 注解调用后端服务（购物车、订单、地址等）。这些服务依赖 `UserContext.getUser()` 获取当前登录用户 ID。但 Agent 的调用链路与前端 HTTP 请求不同，导致 `UserContext` 中缺少用户信息。

---

## 二、核心问题：Agent 调用时 UserContext 无用户信息

### 问题描述

Agent 调用 `CartTool.addToCart()`、`OrderTool.createOrder()` 等方法时，`CartServiceImpl.addItem2Cart()` 中 `UserContext.getUser()` 返回 `null`，导致无法确定当前用户。

### 原因分析

- 前端 HTTP 请求通过 `LoginInterceptor` 解析 token 并设置 `UserContext`
- Agent 调用 `@Tool` 方法时，不走 `LoginInterceptor`，`UserContext` 为空
- 之前 `CartServiceImpl` 中有一个 `userId = 0` 的默认值逻辑（已被注释掉），导致所有 Agent 操作都使用 userId=0

### 解决方案

**1. 常量定义** (`Constant.java`)
```java
public static final String USER_ID = "userId";
```

**2. 请求 DTO 添加 userId** (`ChatDTO.java`)
```java
private Long userId;
```

**3. Controller 传递 userId** (`ChatController.java`)
- 从请求体 `ChatDTO` 获取 `userId`
- 从请求头 `authorization` 解析 token 获取 `userId`
- 将 `userId` 传递给 Service

**4. Service 将 userId 放入 toolContext** (`ChatServiceImpl.java`)
```java
.toolContext(Map.of(
    Constant.REQUEST_ID, requestId,
    Constant.USER_ID, userId != null ? userId : 0L
))
```

**5. Tool 类从 toolContext 提取 userId** (`CartTool.java` / `OrderTool.java` / `AddressTool.java`)
```java
private void setUserFromContext(ToolContext toolContext) {
    Object userIdObj = toolContext.getContext().get(Constant.USER_ID);
    if (userIdObj instanceof Number && ((Number) userIdObj).longValue() > 0) {
        UserContext.setUser(((Number) userIdObj).longValue());
    } else {
        throw new RuntimeException("请先登录后再操作");
    }
}
```

**6. 前端传递 token 和 userId** (`agent-chat.js`)
```javascript
const token = sessionStorage.getItem('token') || '';
const userInfoStr = sessionStorage.getItem('user-info');
// 解析 userId
headers['authorization'] = token;
body: JSON.stringify({ question, sessionId, userId })
```

**7. Nginx 传递 Authorization 头** (`nginx.conf`)
```nginx
proxy_set_header Authorization $http_authorization;
```

## 三、购物车页面显示静态数据问题

### 问题描述

通过 Agent 添加商品到购物车后，刷新购物车页面，显示的是静态的 Apple Macbook 数据（"已选择0件商品"、"总价0.00"），而不是新添加的商品。

### 原因分析

`cart.html` 中有一段**静态 HTML 写死的示例数据**（`.deled` 区域），当购物车请求失败或返回空数据时，这段静态 HTML 就会显示出来。

### 解决方案

- 增加 axios 超时时间（`common.js` 中 `timeout: 2000` → `timeout: 10000`）
- 确认后端服务 `localhost:8080` 已启动
- 重新登录获取有效 token

## 四、JSON.parse(spec) 报错问题

### 问题描述

购物车页面和结算页报错：
```
SyntaxError: Unexpected token '颜', "颜色-银色 / 尺寸-金色" is not valid JSON
```

### 原因分析

- `cart.html` 中使用 `JSON.parse(c.spec)` 解析商品规格
- `order-confirm.html` 中使用 `JSON.parse(str)` 解析商品规格
- 通过 Agent 添加的商品，`spec` 字段值为 `"颜色-银色 / 尺寸-金色"`（非 JSON 格式的字符串）
- 而数据库中其他商品的 `spec` 是 `{"颜色": "银色", "选择内存": "128GB"}`（JSON 格式）
- 当 `JSON.parse("颜色-银色 / 尺寸-金色")` 执行时，抛出 `SyntaxError`，导致整个 Vue 渲染崩溃

### 解决方案

将 `JSON.parse` 替换为安全的 `parseSpec` / `parseJson` 方法，支持两种格式：

```javascript
parseSpec(spec) {
    if (!spec) return {};
    try {
        return JSON.parse(spec);  // JSON 格式
    } catch (e) {
        // 非 JSON 格式：按分隔符解析
        if (typeof spec === 'string' && spec.includes('-')) {
            const parts = spec.split(' / ');
            const result = {};
            parts.forEach(part => {
                const idx = part.indexOf('-');
                if (idx > 0) {
                    result[part.substring(0, idx)] = part.substring(idx + 1);
                } else {
                    result['规格'] = part;
                }
            });
            return result;
        }
        return {};
    }
}
```

**修改的文件**：
- `cart.html`：`JSON.parse(c.spec)` → `parseSpec(c.spec)`
- `order-confirm.html`：`JSON.parse(str)` → `parseJson(str)`（增强版）
