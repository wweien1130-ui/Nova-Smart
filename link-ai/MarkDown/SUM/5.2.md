# 商品搜索工具开发总结

## 1. 问题背景

在开发基于 Spring AI 的智能购物助手时，需要实现一个商品搜索功能，让 AI 能够：
- 根据关键词搜索商品
- 查询商品详情
- 在聊天界面展示商品信息（包括图片）

## 2. 核心问题与解决方案

### 2.1 DTO/VO/PO/Query 的概念区分

**问题**：不清楚 `dto`、`vo`、`po`、`query` 文件夹的含义和作用

**解决方案**：

| 类型 | 含义 | 用途 |
|------|------|------|
| **PO** (Persistent Object) | 持久化对象 | 对应数据库表结构，用于 ORM 映射 |
| **DTO** (Data Transfer Object) | 数据传输对象 | API 接口的请求和响应数据传输 |
| **VO** (Value Object) | 值对象 | 返回给前端的视图对象，只包含显示字段 |
| **Query** | 查询对象 | 封装分页查询和条件筛选参数 |

**架构层次**：
```
PO (数据库实体) → Service → DTO (API传输) → VO (前端展示)
```

### 2.2 工具方法返回类型的选择

**问题**：在 `ItemSearchTool` 中应该引入 `Item.java` (PO) 还是 `ItemPageQuery.java` (Query)？

**解决方案**：
- **PO (Item)**：用于数据库查询和业务逻辑
- **Query (ItemPageQuery)**：用于封装查询条件，不适合作为返回值
- **结论**：使用 PO 进行数据查询，转换为专门的 DTO/VO 返回

### 2.3 创建独立的 ItemInfo 类

**问题**：是否应该修改原始的 `ItemDTO`，还是创建新的类？

**解决方案**：创建独立的 `ItemInfo.java`

```java
@Data
@Builder
public class ItemInfo {
    private Long id;           // 商品ID
    private String name;       // 商品名称
    private double price;      // 价格（元）
    private String image;      // 商品图片URL
    private String category;   // 商品分类
    private String brand;      // 商品品牌
    private String spec;       // 商品规格
}
```

**设计原则**：
- ✅ 符合开闭原则，不破坏原有代码
- ✅ 模块独立，`agent` 模块不需要了解 `hm-service` 的内部结构
- ✅ 可扩展性强，可随时添加新字段

### 2.4 PO 到 ItemInfo 的转换

**问题**：如何将 `Item` (PO) 转换为 `ItemInfo`？

**解决方案**：使用 `Optional.ofNullable` 进行流式转换

```java
public ItemInfo getItemDetail(Long itemId) {
    return Optional.ofNullable(itemId)
            .map(id -> itemService.getById(id))
            .map(item -> ItemInfo.builder()
                    .id(item.getId())
                    .name(item.getName())
                    .price(item.getPrice() / 100.0)  // 分转元
                    .image(item.getImage())
                    .category(item.getCategory())
                    .brand(item.getBrand())
                    .spec(item.getSpec())
                    .build())
            .orElse(null);
}
```

**Optional.ofNullable 的含义**：
- 包装可能为 `null` 的值
- 避免空指针异常
- 支持链式调用 `map()` 和 `filter()`

### 2.5 ToolResultHolder 工具结果存储

**问题**：如何让 ChatService 能够访问工具调用的结果？

**解决方案**：使用 `ToolResultHolder` 存储工具结果

**工具方法中的存储**：
```java
.map(itemInfo -> {
    String field = String.format("%s_%d", ItemInfo.class.getSimpleName(), itemInfo.getId());
    String requestId = String.valueOf(toolContext.getContext().get("requestId"));
    ToolResultHolder.put(requestId, field, itemInfo);
    return itemInfo;
})
```

**ToolResultHolder 实现**：
```java
public class ToolResultHolder {
    private static final Map<String, Map<String, Object>> HANDLER_MAP = new ConcurrentHashMap<>();
    
    public static void put(String key, String field, Object result) {
        HANDLER_MAP.computeIfAbsent(key, k -> new HashMap<>()).put(field, result);
    }
    
    public static Map<String, Object> get(String key) {
        return HANDLER_MAP.get(key);
    }
    
    public static void remove(String key) {
        HANDLER_MAP.remove(key);
    }
}
```

### 2.6 ChatServiceImpl 监听工具调用结果

**问题**：如何在 `ChatServiceImpl` 中监听工具调用并将结果返回给前端？

**解决方案**：使用 `Flux.defer()` 监听 `ToolResultHolder`

```java
.concatWith(Flux.defer(() -> {
    var map = ToolResultHolder.get(requestId);
    if (CollUtil.isNotEmpty(map)) {
        ToolResultHolder.remove(requestId);
        
        var chatEventVO = ChatEventVo.builder()
                .eventData(map)
                .eventType(ChatEventTypeEnum.PARAM.getValue())  // 1003
                .build();
        
        return Flux.just(chatEventVO, STOP_EVENT);
    }
    return Flux.just(STOP_EVENT);
}))
```

**事件类型定义**：
- `1001 (DATA)`：普通文本数据
- `1002 (STOP)`：停止事件
- `1003 (PARAM)`：参数数据（工具调用结果）

### 2.7 前端处理 PARAM 事件

**问题**：前端收到 `eventType === 1003` 的数据后如何展示商品信息？

**解决方案**：在前端添加商品卡片渲染逻辑

```javascript
} else if (event.eventType === 1003) {
    const itemData = event.eventData;
    console.log('收到商品数据:', itemData);
    
    let productCards = '';
    for (const [key, item] of Object.entries(itemData)) {
        if (item && item.image) {
            productCards += `
                <div class="product-card" style="margin-top: 12px; border: 1px solid #e8e8e8; border-radius: 12px; padding: 12px; background: #f9fafb;">
                    <div style="display: flex; gap: 12px;">
                        <img src="${item.image}" alt="${item.name || '商品图片'}" style="width: 120px; height: 120px; object-fit: cover; border-radius: 8px; flex-shrink: 0;">
                        <div style="flex: 1; min-width: 0;">
                            <div style="font-weight: 600; font-size: 16px; color: #1a1a2e; margin-bottom: 4px;">${item.name || '商品名称'}</div>
                            <div style="font-size: 18px; color: #ef4444; font-weight: 600; margin-bottom: 8px;">¥${item.price || '0.00'}</div>
                            <div style="font-size: 12px; color: #6b7280; margin-bottom: 4px;">${item.category || ''} ${item.brand ? '| ' + item.brand : ''}</div>
                            <div style="font-size: 12px; color: #6b7280;">${item.spec || ''}</div>
                        </div>
                    </div>
                </div>
            `;
        }
    }
    
    if (productCards) {
        chat.messages[msgIndex].productCards = productCards;
        let cardsContainer = bubble.nextElementSibling;
        if (!cardsContainer || !cardsContainer.classList.contains('product-cards-container')) {
            cardsContainer = document.createElement('div');
            cardsContainer.className = 'product-cards-container';
            cardsContainer.style.marginTop = '12px';
            bubble.parentNode.insertBefore(cardsContainer, bubble.nextSibling);
        }
        cardsContainer.innerHTML = productCards;
        scrollToBottom();
    }
}
```

### 2.8 图片大小自适应

**问题**：商品图片太大，破坏了聊天界面的布局

**解决方案**：添加 CSS 样式限制图片大小

```css
.message.assistant .bubble img {
    max-width: 100%;
    max-height: 300px;
    width: auto;
    height: auto;
    object-fit: contain;
    border-radius: 8px;
    margin: 8px 0;
    display: block;
}
```

**样式效果**：
- 宽度不超过容器宽度
- 高度不超过 300px
- 保持原始宽高比
- 不会破坏文本格式

```
┌─────────────┐
│   前端提问   │

       ↓
┌─────────────────────────────────────────┐
│          Spring AI ChatClient            │
│   system prompt + user question          │
└──────┬──────────────────────────────────┘
       ↓
┌─────────────────────────────────────────┐
│         ItemSearchTool.searchItems()     │
│   1. 查询数据库 (Item.java)              │
│   2. 转换为 ItemInfo                     │
│   3. 存储到 ToolResultHolder            │
└──────┬──────────────────────────────────┘
       ↓
┌─────────────────────────────────────────┐
│        ChatServiceImpl.concatWith()      │
│   1. 监听 ToolResultHolder              │
│   2. 生成 PARAM 事件 (1003)             │

└──────┬──────────────────────────────────┘
       ↓
┌─────────────────────────────────────────┐
│            前端渲染                       │
│   1. 解析 eventType === 1003            │
│   2. 生成商品卡片 HTML                   │
│   3. 显示商品图片、名称、价格等           │
└─────────────────────────────────────────┘
```

## 5. 关键设计模式

1. **开闭原则**：创建新的 ItemInfo，不修改原有代码
2. **Optional 模式**：安全处理空值
3. **工具上下文**：通过 ToolContext 传递请求 ID
4. **事件驱动**：通过不同事件类型区分数据
5. **响应式编程**：使用 Flux 进行流式处理
