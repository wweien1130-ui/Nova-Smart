# 微服务架构演进流程

## 📋 概述

本文档记录当前项目（AI 智能购物商城）的微服务架构演进流程，包括：
- Feign / OpenFeign 引入
- Spring Cloud Gateway 网关引入
- Sentinel 限流熔断引入
- Kafka 异步消息引入

---

## 🎯 当前架构

```
┌─────────────────────────────────────────────────────┐
│                     前端 / App                      │
└─────────────────────────┬───────────────────────────┘
                          │
                          ▼
                 ┌──────────────────┐
                 │      Nginx       │
                 │  (反向代理+负载)  │
                 └────────┬─────────┘
                          │
                          ▼
              ┌───────────────────────┐

              │    (商城核心服务)       │

              │  └── AddressService    │
```

**问题**：
- Agent 模块与 hm-service 强耦合（同进程部署）
- 无法独立部署、独立扩缩容
- 缺乏限流、熔断保护

## Phase 1：Agent 独立部署 + Feign 引入

### 1.1 目标

```
改造前：
agent ──依赖──> hm-service（同进程）

改造后：
agent（独立进程） ──Feign──> hm-service（独立进程）
```

### 1.2 改动清单

#### 1.2.1 Agent pom.xml 新增依赖

```xml
<!-- OpenFeign 服务调用 -->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>

<!-- Nacos 服务发现 -->
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
</dependency>

<!-- 负载均衡 -->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>
```

#### 1.2.2 启动类添加 @EnableFeignClients

```java
@SpringBootApplication
@EnableFeignClients  // 新增
public class AgentApplication {
    public static void main(String[] args) {
        SpringApplication.run(AgentApplication.class, args);
    }
}
```

#### 1.2.3 创建 Feign Client 接口

```java
    List<ItemDTO> getItemsByIds(@RequestParam("ids") String ids);

```


#### 1.2.4 Tool 层改造

**改动前**：
```java
@Component
@RequiredArgsConstructor
public class ItemSearchTool {
    private final IItemService itemService;  // 直接注入 Service
}
```

**改动后**：
```java
@Component
@RequiredArgsConstructor
public class ItemSearchTool {
    private final ItemFeignClient itemFeignClient;  // 改用 Feign
}
```

#### 1.2.5 application.yaml 配置

```yaml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: 192.168.x.x:8848  # Nacos 地址
        namespace: dev
        group: HMAIL_GROUP

# 服务名称（Agent 注册到 Nacos 的名字）
feign:
  client:
    config:
      default:
        connect-timeout: 5000
        read-timeout: 10000
```

#### 1.2.6 hm-service 无需改动

hm-service 本身就是 REST API，只要 Controller 接口正常暴露即可。

### 2.5 路由配置 application.yaml

```yaml
server:
  port: 80

spring:
  application:
    name: gateway
  cloud:
    nacos:
      discovery:
        server-addr: 192.168.x.x:8848
        namespace: dev
        group: HMAIL_GROUP

    gateway:
      routes:
        # Agent 服务路由
        - id: agent-route
          uri: lb://agent-service
          predicates:
            - Path=/agent/**
          filters:
            - StripPrefix=1

        # 商品服务路由
        - id: item-route
          uri: lb://hmall-service
          predicates:
            - Path=/items/**
          filters:
            - StripPrefix=1

        # 购物车服务路由
        - id: cart-route
          uri: lb://hmall-service
          predicates:
            - Path=/carts/**
          filters:
            - StripPrefix=1

        # 订单服务路由
        - id: order-route
          uri: lb://hmall-service
          predicates:
            - Path=/orders/**
          filters:
            - StripPrefix=1

        # 地址服务路由
        - id: address-route
          uri: lb://hmall-service
          predicates:
            - Path=/addresses/**
          filters:
            - StripPrefix=1

      # 全局跨域配置
      globalcors:
        cors-configurations:
          '[/**]':
            allowedOriginPatterns: "*"
            allowedMethods:
              - GET
              - POST
              - PUT
              - DELETE
              - OPTIONS
            allowedHeaders: "*"
            allowCredentials: true
            maxAge: 3600
```

## Phase 3：Sentinel 限流熔断

### 3.1 目标

```
- 保护 LLM 调用（防止超时雪崩）
- 保护核心接口（防止刷接口）
- 熔断降级（服务不可用时快速失败）
```

### 3.2 Sentinel 架构说明

```
┌──────────────┐     ┌──────────────────┐
│  Sentinel    │◄────│  Dashboard       │
│  客户端       │     │  (控制台)        │
│  (嵌入各服务) │     │  192.168.x.x:8858│
└──────────────┘     └──────────────────┘
       ▲
       │
┌──────┴──────┐
│ 各微服务     │
│ - Gateway   │
│ - Agent     │
│ - hm-service│
└─────────────┘
```

### 3.7 熔断降级配置（代码层面）

```java
// Agent 启动类添加
@Bean
public SentinelFallbackHandler sentinelFallbackHandler() {
    return (exchange, ex) -> {
        // 降级返回
        ServerHttpResponse response = exchange.getResponse();
        response.setStatusCode(HttpStatus.SERVICE_UNAVAILABLE);
        return response.setComplete();
    };
}
```

```

                     │  │ user-action │    │
```

### 4.6 消息发送代码

```java
// com.liang.agent.messaging.ChatLogProducer.java
@Component
@RequiredArgsConstructor
public class ChatLogProducer {
    
    private final KafkaTemplate<String, String> kafkaTemplate;
    
    public void sendChatLog(String sessionId, String userMessage, String aiResponse) {
        Map<String, Object> log = new HashMap<>();
        log.put("sessionId", sessionId);
        log.put("userMessage", userMessage);
        log.put("aiResponse", aiResponse);
        log.put("timestamp", System.currentTimeMillis());
        
        String json = JSONUtil.toJsonStr(log);
        kafkaTemplate.send("chat-log", sessionId, json);
    }
}
```

### 4.7 消息消费代码（可选，单独模块或 hm-service）

```java
// com.hmall.consumer.ChatLogConsumer.java
@Component
@RequiredArgsConstructor
public class ChatLogConsumer {
    
    private final ChatLogMapper chatLogMapper;
    
    @KafkaListener(topics = "chat-log", groupId = "chat-log-consumer")
    public void consumeChatLog(String message) {
        Map<String, Object> log = JSONUtil.toBean(message, Map.class);
        
        ChatLog chatLog = new ChatLog();
        chatLog.setSessionId((String) log.get("sessionId"));
        chatLog.setUserMessage((String) log.get("userMessage"));
        chatLog.setAiResponse((String) log.get("aiResponse"));
        chatLog.setCreateTime(new Date());
        
        chatLogMapper.insert(chatLog);
    }
}
```

### 4.8 在 Agent 中集成消息发送

在 `ChatServiceImpl` 中添加消息发送：

```java
@RequiredArgsConstructor
public class ChatServiceImpl implements ChatService {
    
    private final ChatLogProducer chatLogProducer;
    
    public void saveMessage(String sessionId, String userMessage, String aiResponse) {
        // ... 保存到 Redis
        
        // 异步发送到 Kafka
        chatLogProducer.sendChatLog(sessionId, userMessage, aiResponse);
    }
}
```

## 🔧 常见问题

### Q1: Feign 调用超时怎么办？

```yaml
feign:
  client:
    config:
      default:
        connect-timeout: 5000   # 连接超时
        read-timeout: 10000     # 读取超时
      hmall-service:
        connect-timeout: 3000
        read-timeout: 15000     # AI 调用可能较慢
```

### Q2: Gateway 路由 404？

检查：
1. 服务是否注册到 Nacos
2. 服务名称是否正确（区分大小写）
3. 路径匹配是否正确（`/agent/**` vs `/agent`）

### Q3: Sentinel 不生效？

检查：
1. Dashboard 是否启动
2. 是否有引入 `sentinel-transport-simple-http` 依赖
3. 是否配置了 `-Dcsp.sentinel.api.port=8719`

### Q4: Kafka 消息丢失？

检查：
1. Producer 配置 `acks=all`
2. Consumer 配置手动提交 offset
3. 确认 Consumer 所属 group 正确
